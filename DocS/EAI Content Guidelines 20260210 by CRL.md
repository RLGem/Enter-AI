│   הצלחה → ביטחון → פתיחות → הבנה עמוקה                    │
└────────────────────────────────────────────────────────────┘

## הכללים הנגזרים

### 1. כל מודול מתחיל בפעולה

```
┌────────────────────────────────────────────────────────────┐
│ ✅ נכון:                                                   │
│ "עכשיו תעשה X... מעולה! עכשיו בוא נבין למה זה עבד"        │
│                                                            │
│ ❌ לא נכון:                                                │
│ "קודם נלמד את התיאוריה של X, ואז תתרגל"                   │
└────────────────────────────────────────────────────────────┘
```

### 2. תיאוריה באה אחרי הצלחה

הלומד קודם מרגיש "אני יכול" ורק אז שומע "ככה זה עובד".

### 3. כל פעולה מסתיימת בתוצר מוחשי

הלומד חייב לסיים עם משהו ביד - לא רק "הבנה".

### 4. 💥 זעזוע קטן בכל פתיחה — "בעיטה באמונה הנאיבית"

```
┌────────────────────────────────────────────────────────────────┐
│ 💥 העיקרון:                                                    │
│                                                                │
│ כל פתיחת שלב או קורס מתחילה ב"זעזוע קטן":                    │
│ סיפור קצר שמראה איך האמונה הנאיבית של היזם                   │
│ לא עומדת במבחן המציאות.                                       │
│                                                                │
│ לא כדי לשבור — כדי לפתוח.                                     │
│ אם היזם לא מרגיש אי-נוחות קלה בפתיחה —                       │
│ הוא לא ילמד שום דבר חדש.                                      │
└────────────────────────────────────────────────────────────────┘
```

**המבנה — 3 חלקים:**

```
┌────────────────────────────────────────────────────────────────┐
│ חלק 1: הסיפור                                                 │
│ ─────────────────                                              │
│ דמות (דני / EventApp) עושה את מה שרוב היזמים עושים.          │
│ זה נשמע הגיוני. זה מרגיש נכון.                                │
│ ואז — זה נכשל. לא כי הוא טיפש,                                │
│ אלא כי האמונה הנאיבית לא עמדה במציאות.                       │
│                                                                │
│ חלק 2: השאלה המתריסה                                          │
│ ──────────────────────                                         │
│ שאלה אחת שמאלצת את היזם להסתכל על עצמו.                      │
│ לא שאלה רטורית. שאלה שהתשובה עליה מפחידה קצת.                │
│ "אם אתה מפחד לבקש כסף על המוצר שלך —                         │
│  מה זה אומר על כמה אתה מאמין שהוא שווה?"                      │
│                                                                │
│ חלק 3: AI כשותף (לא כמושיע)                                   │
│ ──────────────────────────────                                  │
│ מה AI יכול לעשות (הכוח-על).                                   │
│ מה AI לא יכול לעשות (ההחלטה, האומץ, השיחה).                   │
│ prompt מוכן שמדגים את השותפות.                                 │
└────────────────────────────────────────────────────────────────┘
```

**אמונות נאיביות נפוצות (לשימוש בפתיחים):**

```
┌────────────────────────────────────────────────────────────────┐
│ 🫧 האמונה הנאיבית          →  💥 המציאות                      │
│ ──────────────────────────────────────────────                  │
│ "אני סבלתי מזה =            "אתה אולי מיעוט.                  │
│  כולם סובלים"                 תשאל אחרים."                     │
│                                                                │
│ "חבר אמר שזה רעיון טוב"     "חברים לא ישקרו לך בפנים.        │
│                                הם פשוט לא יגידו את האמת."      │
│                                                                │
│ "צריך לבנות את כל             "הלקוח רוצה דבר אחד             │
│  הפיצ'רים שתכננתי"            שעובד. לא 12 שלא."              │
│                                                                │
│ "מחיר נמוך = יותר לקוחות"    "מחיר נמוך = לקוחות שלא          │
│                                מעריכים את מה שאתה בונה."       │
│                                                                │
│ "100 משתמשים בחינם            "100 בחינם = תחביב.               │
│  עדיף מ-5 משלמים"             5 משלמים = עסק."                 │
│                                                                │
│ "אני צריך להיות בכל           "אתה צריך ערוץ אחד               │
│  הרשתות החברתיות"              שעובד. לא 7 בינוניים."          │
│                                                                │
│ "BMC יעזור לי להבין           "BMC לפני לקוחות = ניחוש.         │
│  את העסק שלי"                  BMC אחרי לקוחות = מפה."         │
│                                                                │
│ "MVP = מוצר מכוער"            "MVP = מוצר שעושה דבר אחד        │
│                                ועושה אותו יפה."                │
│                                                                │
│ "אני צריך לדעת קוד            "אתה צריך לדעת מה לבקש           │
│  כדי לבנות מוצר"               מה-AI. הוא יודע קוד."           │
│                                                                │
│ "הרבה ביקורים =               "הרבה ביקורים + המרה נמוכה       │
│  הצלחה"                        = כסף זורק לפח."                │
│                                                                │
│ "משקיע ירצה לשמוע             "משקיע רוצה לשמוע מספרים.        │
│  על החזון שלי"                 חזון בלי Data = סיפור."          │
└────────────────────────────────────────────────────────────────┘
```

**כללי כתיבה לזעזוע:**

```
┌────────────────────────────────────────────────────────────────┐
│ ✅ כן:                                                         │
│ • ספציפי — שם, מספר, תרחיש מוחשי                              │
│ • אמפתי — "זה לא טיפשי, רוב היזמים עושים את זה"             │
│ • בונה — הזעזוע מוביל ישר לפעולה                              │
│ • קצר — 4-6 משפטים לסיפור, משפט אחד לשאלה                    │
│                                                                │
│ ❌ לא:                                                         │
│ • מתנשא — "ברור שזה לא עובד"                                  │
│ • מדכא — "רוב הסטארטאפים נכשלים"                             │
│ • מופשט — "חשוב לחשוב אסטרטגית"                              │
│ • ארוך — אם הסיפור מעל 8 משפטים, קצר                         │
│                                                                │
│ 🎯 המבחן:                                                     │
│ "אחרי שהיזם קורא את הפתיח — הוא מרגיש:                       │
│  'רגע, אולי אני עושה את זה לא נכון'                           │
│  ולא: 'אני מפסיד, אין טעם'"                                   │
└────────────────────────────────────────────────────────────────┘
```
