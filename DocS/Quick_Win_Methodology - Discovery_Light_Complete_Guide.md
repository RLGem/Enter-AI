# Discovery Light - המדריך המלא
## איך לעשות 5-10 שיחות שיפתחו לך את העיניים

---

## תשובה קצרה לשאלה שלך:

**לא לשאול אנשים רנדומליים "אילו בעיות יש לכם?"**

זה יהיה disaster! 🚫

### הדרך הנכונה:

1. **התחל מתחום** שאתה מכיר/מעניין אותך
2. **מצא אנשים** שנמצאים בתחום הזה
3. **שאל על החיים שלהם** בתחום הזה
4. **גלה בעיות** שהם חווים (לא תציע פתרונות!)
5. **תעד תובנות**

---

## שני סוגי Discovery

### תרחיש A: יש לך רעיון/תחום כללי ✅

**דוגמה:** "אני רוצה לעזור לאנשים עם תכנון אירועים"

```
יש לך → תחום: תכנון אירועים
       ↓
צריך → למצוא את הבעיה הספציפית שתפתור
```

**מי לשאול?**
- אנשים שתכננו אירועים לאחרונה
- מתכנני אירועים מקצועיים
- אנשים שמתכננים אירוע עכשיו

**מה לשאול?**
- ספר לי על התהליך של תכנון האירוע
- מה היה הכי מאתגר?
- איך התמודדת עם X?
- מה היית משנה בפעם הבאה?

### תרחיש B: אין לך רעיון, מחפש בעיה ❓

**דוגמה:** "אני רוצה להקים סטארטאפ אבל אין לי רעיון"

```
צריך → למצוא תחום שאתה מכיר/מתחבר אליו
       ↓
אז → למצוא בעיה בתחום הזה
```

**איך מתחילים?**
1. **רשום תחומים** שאתה מכיר:
   - עבודה שלך
   - תחביבים
   - frustrations אישיות
   - דברים שאתה עושה הרבה

2. **בחר תחום אחד**

3. **מצא אנשים בתחום**

4. **למד על החיים שלהם**

## The Mom Test - הכלל הזהב

### מה זה The Mom Test?

ספר מאת Rob Fitzpatrick שמסביר **איך לעשות Customer Interviews נכון**.

### הכלל העיקרי:

> **"אל תשאל את אמא שלך אם הרעיון שלך טוב"**

למה? כי היא תאמר "כן מותק, זה נפלא!" גם אם זה נורא.

### 3 הכללים:

#### 1. דבר על החיים שלהם, לא על הרעיון שלך ✅
```
❌ רע: "האם תשתמש באפליקציה לתכנון אירועים?"
✅ טוב: "ספר לי איך תכננת את החתונה שלך"

❌ רע: "האם תשלם 50₪ לחודש עבור זה?"
✅ טוב: "כמה הוצאת על כלי ניהול פרויקט?"
```

#### 2. שאל על התנהגות עבר, לא על עתיד ✅
```
❌ רע: "האם תשתמש ב-X אם נבנה אותו?"
✅ טוב: "מה עשית בפעם האחרונה שנתקלת ב-Y?"

❌ רע: "האם תקנה את המוצר שלנו?"
✅ טוב: "מה קנית כדי לפתור את הבעיה הזו?"
```

#### 3. דבר פחות, שמע יותר ✅
```
אתה: 20% מהשיחה
הם: 80% מהשיחה

אתה שואל → הם מספרים → אתה שואל follow-up
```

## המבנה המדויק של Discovery Light

### לפני השיחות (1-2 שעות הכנה)

#### שלב 1: בחר תחום
**שאלות לעצמך:**
- מה אני מכיר?
- מה מעניין אותי?
- איפה אני רואה frustration?

**דוגמאות:**
- תכנון אירועים (אם עשית זאת)
- כושר/תזונה (אם מתאמן)
- עבודה מהבית (אם עובד remote)
- לימודים/חינוך (אם אתה מורה/סטודנט)

#### שלב 2: הגדר קהל יעד ראשוני

**לא כללי!** ספציפי!

```
❌ רע: "אנשים שצריכים לארגן דברים"
✅ טוב: "אנשים שתכננו חתונה בשנה האחרונה"

❌ רע: "אנשים שרוצים להתאמן"
✅ טוב: "אנשים שהצטרפו לחדר כושר אבל הפסיקו אחרי חודש"

❌ רע: "סטודנטים"
✅ טוב: "סטודנטים לתואר שני שעובדים במקביל"
```

#### שלב 3: מצא 5-10 אנשים

**איפה מוצאים?**
1. **חוג מכרים**
   - חברים
   - משפחה (אבל היזהר מ-Mom Test!)
   - עמיתים לעבודה
   - שכנים

2. **רשתות חברתיות**
   - קבוצות פייסבוק רלוונטיות
   - LinkedIn
   - קהילות Discord/Slack

3. **מקומות פיזיים**
   - בתי קפה
   - אירועים
   - meetups

4. **פלטפורמות**
   - Reddit
   - קהילות מקוונות

**איך ניגשים?**
```
הודעה לדוגמה:

"היי [שם],

אני עושה מחקר קטן על [תחום] ושמעתי ש[עשית X לאחרונה].

אשמח ל-20 דקות שיחה לשמוע על החוויה שלך.

זה לא מכירה - רק רוצה ללמוד :)

פנוי/ה השבוע?

תודה!
[שמך]"
```

#### שלב 4: הכן שאלות (Interview Script)

**מבנה כללי (20-30 דקות):**

```
1. פתיחה (2 דקות)
   - הכרות
   - הסבר: "אני לומד על X, לא מוכר כלום"
   
2. Context (5 דקות)
   - ספר לי על עצמך
   - איך הגעת ל-[תחום]?
   
3. Deep Dive (15 דקות)
   - שאלות ספציפיות על התחום
   - בעיות
   - פתרונות נוכחיים
   
4. Wrap-up (3 דקות)
   - יש משהו שלא שאלתי?
   - מכיר עוד מישהו שאוכל לדבר איתו?
```

## דוגמה מלאה: EventApp Discovery

### תחום: תכנון אירועים (חתונות)

#### קהל יעד:
**"זוגות שתכננו חתונה ב-12 החודשים האחרונים"**

#### Interview Script:

**פתיחה:**
```
"היי [שם], תודה שהסכמת לדבר איתי!

אני עושה מחקר על תכנון אירועים, בעיקר חתונות.
זה לא מכירה - פשוט רוצה ללמוד מהחוויה שלך.

אני אשאל שאלות, ואתה פשוט תספר באופן חופשי.
אין תשובות נכונות או לא נכונות :)

זה OK?"
```

**Context Questions:**
```
1. ספר לי איך הכל התחיל - איך התחלתם לתכנן?
2. מה היה הצעד הראשון?
3. מי היה מעורב בתכנון? (את/ה לבד? בן/בת זוג? הורים?)
```

**Deep Dive - Problems:**
```
4. מה היה הדבר הכי מאתגר בתהליך?
   [תן להם לדבר... תעד!]
   
5. ספר לי על רגע שהרגשת overwhelmed?
   [הם יספרו סיפור - זה זהב!]
   
6. איך עקבתם אחרי כל המשימות?
   [Excel? נייר? בראש?]
   
7. איך תיאמתם עם הספקים? (צלם, אולם, קייטרינג...)
   [מיילים? טלפונים? WhatsApp?]
   
8. האם שכחתם משהו חשוב?
   [אם כן - למה? איך זה קרה?]
   
9. איך ניהלתם את התקציב?
   [Excel? אפליקציה? בראש?]
   
10. מה הכי הלחיץ אותך? למה?
```

**Deep Dive - Current Solutions:**
```
11. האם השתמשתם בכלים כלשהם? (אפליקציות, ממשקים)
    [אם כן - איזה? למה בחרתם בהם?]
    [אם לא - למה לא?]
    
12. מה עבד טוב בכלים שהשתמשתם?
13. מה לא עבד?
14. כמה שילמתם על הכלים? (אם בכלל)
```

**Money Questions (חשוב!):**
```
15. אם היה כלי שפותר את [הבעיה שהזכירו],
    כמה זה היה שווה לך?
    
    [לא לשאול ישירות! לתת context:]
    "בהשוואה לדברים אחרים שקנית לחתונה..."
```

**Wrap-up:**
```
16. אם היית עושה את זה שוב, מה היית משנה?
17. איזה עצה היית נותן למישהו שמתחיל לתכנן עכשיו?
18. יש משהו שלא שאלתי שחשוב לדעת?
19. מכיר עוד זוגות שתכננו חתונה לאחרונה?
    [snowball sampling - קבל עוד אנשים!]
```

## תיעוד השיחה - קריטי!

### במהלך השיחה:

**רשום הכל!** (או הקלט - אם הם מאשרים)

```
נושא: תכנון חתונה
מרואיין: שרה, 28, תכננה חתונה לפני 6 חודשים
תאריך: 15.12.24

[רשימות:]

בעיות שהזכירה:
- "הרגשתי שיש לי מיליון משימות ולא ידעתי מאיפה להתחיל"
- "שכחנו להזמין פרחים - זה היה disaster"
- "כל ספק דיבר איתנו ב-WhatsApp אחר - זה היה מבולגן"
- "לא ידעתי איפה אנחנו עם התקציב עד שהיה מאוחר מדי"

כלים ששימשו:
- Excel לרשימת משימות (לא עבד - "מבולגן מדי")
- Google Calendar (עבד OK)
- WhatsApp לתקשורת עם ספקים (מבולגן)

ציטוטים חשובים:
- "הייתי משלמת 100₪ לחודש אם זה היה חוסך לי את הלחץ"
- "הדבר שהכי חיסר לי זה checklist פשוטה"

רגשות:
- overwhelmed, מתוח, chaos
- "זה היה אמור להיות כיף אבל זה היה nightmare"
```

### אחרי השיחה:

**סיכום מיידי (5 דקות):**
```
Top 3 Insights:
1. אנשים מרגישים overwhelmed מרוב משימות
2. כלים קיימים (Excel) לא מספיקים
3. תקשורת עם ספקים מבולגנת
```

## אחרי 5-10 שיחות - ניתוח

### שלב 1: Affinity Mapping

**כל הבעיות שנאמרו:**
```
שיחה 1: overwhelmed, שכח פרחים, WhatsApp מבולגן
שיחה 2: Excel לא עבד, תקציב לא ברור, לחץ
שיחה 3: לא ידע מה לעשות ראשון, איבד קובץ
שיחה 4: Communicate עם ספקים קשה, overwhelmed
שיחה 5: שכח לשלם לספק, תקציב התפוצץ
...
```

**קבץ לקטגוריות:**
```
קטגוריה A: Task Management
- overwhelmed מרוב משימות (4 פעמים)
- לא ידע מאיפה להתחיל (3 פעמים)
- שכח דברים (3 פעמים)

קטגוריה B: Communication
- WhatsApp מבולגן (4 פעמים)
- קשר עם ספקים scattered (5 פעמים)

קטגוריה C: Budget
- תקציב לא ברור (4 פעמים)
- הוצאות התפוצצו (2 פעמים)

קטגוריה D: Tools
- Excel לא עובד (5 פעמים!)
- אפליקציות קיימות יקרות מדי (2 פעמים)
```

### שלב 2: Pattern Recognition

**מה חוזר?**
```
TOP 3 PROBLEMS:
1. 🔥 Task Management - אנשים overwhelmed
   (8/10 אנשים הזכירו!)
   
2. 🔥 Excel doesn't work
   (7/10 אנשים!)
   
3. 🔥 Communication with suppliers
   (6/10 אנשים!)
```

### שלב 3: Validate Problem-Solution Fit

**האם יש בעיה אמיתית?**
```
✅ כן, אם:
- 70%+ מהאנשים מזכירים בעיה דומה
- הם עושים workarounds (Excel, נייר)
- הם מבטאים frustration חזק
- הם מוכנים לשלם (אמרו סכום)

❌ לא, אם:
- כל אחד מזכיר בעיה שונה
- אנשים אומרים "זה OK, לא באמת מפריע"
- אף אחד לא מוכן לשלם
- אין פתרונות נוכחיים בשימוש
```

### שלב 4: Problem Statement

**כתוב את זה בצורה ברורה:**

```
Problem Statement - EventApp:

"זוגות שמתכננים חתונה מרגישים overwhelmed 
מכמות המשימות, הספקים, והתקציב.

כלים קיימים כמו Excel מסורבלים ולא מספיקים.

התקשורת עם ספקים מפוזרת על פני WhatsApp, 
מיילים, וטלפונים - מה שגורם לשכחות ולחצים.

אנשים מוכנים לשלם 50-100₪ לחודש עבור כלי 
שיעשה סדר בכל התהליך."
```

## דוגמה שנייה: תחום אחר (כושר)

### תרחיש:
**"אני רוצה לעזור לאנשים עם כושר, אבל לא יודע בדיוק מה הבעיה"**

### קהל יעד:
**"אנשים שהצטרפו לחדר כושר אבל הפסיקו אחרי 1-3 חודשים"**

### Interview Script מקוצר:

```
Context:
1. ספר לי מתי הצטרפת לחדר כושר
2. מה היה המוטיבציה?
3. איך היו החודשים הראשונים?

Problems:
4. מתי התחלת להגיע פחות?
5. מה קרה? (תן להם לספר!)
6. מה היה מונע ממך להגיע?
7. האם ניסית לחזור? מה קרה?

Current Solutions:
8. האם השתמשת באפליקציות? אילו?
9. האם היה לך מאמן אישי?
10. מה עזר? מה לא?

Money:
11. כמה שילמת על מנוי לחודש?
12. האם היית משלם יותר אם זה היה מבטיח שתמשיך?
```

### תוצאות אפשריות (דוגמה):

```
אחרי 10 שיחות:

Pattern #1: Motivation
- "התחלתי מוטיבציה גבוהה אבל ירדתי אחרי שבועיים"
- "לא היה לי אף אחד לדחוף אותי"
- "שכחתי את המטרות שלי"

Pattern #2: Planning
- "לא ידעתי מה לעשות בחדר הכושר"
- "הייתי לוקח תוכנית מהאינטרנט אבל לא עקבתי"

Pattern #3: Social
- "הייתי מגיע לבד - זה משעמם"
- "חברים הפסיקו אז גם אני"

→ אולי הפתרון הוא אפליקציה ל-accountability ו-social workout?
```

## שגיאות נפוצות ב-Discovery Light

### ❌ שגיאה #1: "האם תשתמש ב...?"

```
אתה: "אני חושב לבנות אפליקציה לתכנון אירועים. 
      האם תשתמש בה?"
      
הם: "כן, נשמע נחמד!" (lie - הם מנומסים)

זה לא validation!
```

**במקום:**
```
אתה: "ספר לי על התהליך של תכנון החתונה"
הם: [מספרים על הכאב האמיתי]
זה validation!
```

### ❌ שגיאה #2: לדבר על הרעיון שלך

```
אתה: "אני בונה אפליקציה עם AI שעושה X, Y, Z..."
הם: "אה... מעניין..." (מנומסים, לא באמת)

זה pitch, לא discovery!
```

**במקום:**
```
אתה: "מה השתמשת כדי לנהל את המשימות?"
הם: "Excel, אבל זה היה מבולגן"
זה תובנה!
```

### ❌ שגיאה #3: לשאול על העתיד

```
אתה: "האם תשלם 50₪ לחודש עבור זה?"
הם: "כן בטח!" (קל לומר כן)

זה לא commitment!
```

**במקום:**
```
אתה: "מה שילמת על כלים לתכנון האירוע?"
הם: "ממ... לא שילמתי על כלום. השתמשתי ב-Excel חינמי"
זה מציאות!
```

### ❌ שגיאה #4: Leading Questions

```
אתה: "הרגשת overwhelmed מכל המשימות, נכון?"
הם: "אה... כן..." (אתה שמת את המילים בפה שלהם)

זה bias!
```

**במקום:**
```
אתה: "איך הרגשת במהלך התהליך?"
הם: "איזה מתח! הרגשתי שאני טובע!"
זה אמיתי!
```

### ❌ שגיאה #5: לא לתעד

```
אחרי השיחה: "היה נחמד... מה בדיוק הם אמרו?"
→ שכחת! זה אבוד!
```

**במקום:**
```
במהלך: כותב/מקליט
אחרי: סיכום מיידי
→ יש לך data!
```

## לסיכום

### Discovery Light ≠ "שאל אנשים רנדומליים מה הבעיה שלהם"

### Discovery Light = תהליך מובנה:

1. ✅ בחר **תחום ספציפי**
2. ✅ מצא **אנשים רלוונטיים** (5-10)
3. ✅ הכן **Interview Script** (The Mom Test)
4. ✅ **שמע** את הסיפורים שלהם (80% הם, 20% אתה)
5. ✅ **תעד** הכל
6. ✅ **נתח** patterns
7. ✅ **כתוב** Problem Statement

**תוצאה:**
→ הבנה עמוקה של בעיה אמיתית
→ בסיס ל-Prototype שיפתור בעיה אמיתית
→ לא בזבוז זמן על משהו שאף אחד לא צריך!
