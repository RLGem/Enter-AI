# **1. רקע על Enter AI והקוריקולום**

## מהו Enter AI?

Enter AI הוא קוריקולום יזמות מקיף המלמד יזמות בעידן ה-AI. הקורסים מיועדים ל:
- יזמים מתחילים
- אנשים לא-טכניים שרוצים לבנות מוצרים
- קבוצות יזמות ולומדים עצמאיים

## הפילוסופיה המרכזית

> **"נעשה ונשמע"** - קודם עושים, אחר כך מבינים

העיקרון המנחה: **Quick Wins ותחושת יכולת כמה שיותר מהר**. הלומדים צריכים להרגיש הצלחה ויכולת מהרגע הראשון, לפני שצוללים לתיאוריה.

## EventApp - הדוגמה המרכזית

בכל הקורסים משתמשים ב-**EventApp** כדוגמה עקבית:
- אפליקציה לתכנון חתונות/אירועים
- מוכרת לכולם (כולם תכננו אירוע)
- עשירה ב-features אפשריים
- מאפשרת המחשה של כל הכלים והמתודולוגיות

**Problem Statement של EventApp:**
> "זוגות מאורסים בישראל בגילאי 25-35 שמתכננים חתונה בפעם הראשונה מתקשים לעקוב אחרי 50+ משימות ו-15+ ספקים לאורך 6-12 חודשים, מה שגורם לשכחת פרטים קריטיים, חריגה מתקציב, ולחץ ומתחים בזוגיות. קריטי עכשיו כי עלות חתונה ממוצעת עלתה ל-180K ואין כלי מקומי מותאם."

# **3. שיטת הפרטוטייפינג - הבסיס התיאורטי**

## הרעיון המרכזי

**Pretotyping** (לא Prototyping!) = "Fake it before you make it"

המטרה: לבדוק אם אנשים **באמת** רוצים את המוצר שלך - **לפני** שמשקיעים זמן וכסף בבנייה.

## ההבדל בין Pretotype ל-Prototype

| | Pretotype | Prototype |
|:--|:----------|:----------|
| **מטרה** | לבדוק אם יש ביקוש | לבדוק אם המוצר עובד |
| **זמן** | שעות עד ימים | שבועות עד חודשים |
| **עלות** | כמעט אפס | משמעותית |
| **מה בודקים** | "האם אנשים רוצים את זה?" | "האם זה עובד טכנית?" |
| **כישלון** | זול ומהיר | יקר וכואב |

## הטעות הנפוצה של יזמים

```
❌ הדרך הלא נכונה:
רעיון → בנייה (חודשים) → השקה → "אף אחד לא רוצה את זה" 😢

✅ הדרך הנכונה:
רעיון → Pretotype (ימים) → בדיקה → לומדים → משפרים/פיבוט → בנייה
```

## עקרונות מפתח

### 1. "Make sure you're building the RIGHT it, before you build it RIGHT"
לפני שבונים את המוצר נכון - לוודא שבונים את המוצר **הנכון**.

### 2. "Your own opinion, although interesting, is irrelevant"
מה שאתה חושב לא משנה. מה שמשנה: מה **הלקוחות עושים** (לא מה הם אומרים).

### 3. "Data beats opinions"
נתונים מנצחים דעות. תמיד.

### 4. "Fail fast, fail cheap"
אם הולכים להיכשל - עדיף מהר וזול מאשר אחרי שנה והרבה כסף.

# **5. FUMES - מסגרת הערכת בעיות**

## מהו FUMES?

FUMES הוא כלי להערכה מהירה האם בעיה שווה לפתור:

| אות | משמעות | שאלה |
|:---:|:--------|:-----|
| **F** | Frequent (תכופה) | כמה פעמים הבעיה קורה? |
| **U** | Urgent (דחופה) | כמה דחוף לפתור? |
| **M** | Money (כסף) | האם ישלמו על פתרון? |
| **E** | Easy to find (קל למצוא) | האם קל למצוא את הלקוחות? |
| **S** | Solvable (פתירה) | האם אפשר באמת לפתור? |

## סיסטם הציונים

- ✅ ירוק = מעולה
- 🟡 צהוב = בסדר
- ❌ אדום = בעייתי

**כלל אצבע:**
- 4-5 ירוקים = 🟢 בעיה מעולה! ממשיכים
- 3 ירוקים = 🟡 יש פוטנציאל, לחקור יותר
- 0-2 ירוקים = 🔴 כדאי לחשוב על בעיה אחרת
