### Database Design בסיסי

```
┌────────────────────────────────────────────────────────────────────────────┐
│ 🗄️ Database Design ל-MVP:                                                 │
│                                                                            │
│ 📊 SQL vs NoSQL:                                                          │
│                                                                            │
│ SQL (PostgreSQL, MySQL) - השתמש כש:                                       │
│ ✅ יש קשרים בין נתונים (Users → Tasks)                                   │
│ ✅ צריך Transactions (כסף, הזמנות)                                       │
│ ✅ צריך לעשות שאילתות מורכבות                                            │
│ ✅ רוב המקרים! (המלצה ברירת מחדל)                                        │
│                                                                            │
│ NoSQL (MongoDB, Firebase) - השתמש כש:                                     │
│ ✅ נתונים לא מובנים (JSON משתנה)                                         │
│ ✅ Scale מטורף נדרש                                                       │
│ ✅ Real-time sync (Firebase)                                              │
│                                                                            │
│ ───────────────────────────────────────────────────────────────────────   │
│                                                                            │
│ 📝 כללי עיצוב:                                                            │
│                                                                            │
│ 1. התחל פשוט - 3-5 טבלאות ל-MVP                                          │
│ 2. תמיד יהיה: id, created_at, updated_at                                 │
│ 3. Normalize בהתחלה, Denormalize כשיש צורך                               │
│ 4. אינדקסים על שדות שמחפשים בהם                                         │
└────────────────────────────────────────────────────────────────────────────┘
```
