## 3.1 🗣️ Customer Discovery - גילוי לקוחות

### The Mom Test - העיקרון

```
┌────────────────────────────────────────────────────────────────────────────┐
│ 📖 The Mom Test (Rob Fitzpatrick):                                        │
│                                                                            │
│ 💡 הרעיון: אמא שלך תמיד תגיד שהרעיון שלך מעולה.                           │
│    לכן - אל תשאל שאלות שאמא יכולה לשקר בהן!                              │
│                                                                            │
│ 🎯 3 הכללים:                                                              │
│                                                                            │
│ 1️⃣ דבר על החיים שלהם, לא על הרעיון שלך                                   │
│    ❌ "מה אתה חושב על הרעיון שלי?"                                        │
│    ✅ "ספר לי על הפעם האחרונה שניסית לעשות X"                             │
│                                                                            │
│ 2️⃣ שאל על העבר, לא על העתיד                                              │
│    ❌ "האם היית משתמש באפליקציה כזו?"                                     │
│    ✅ "איך פתרת את הבעיה הזו בפעם האחרונה?"                               │
│                                                                            │
│ 3️⃣ דבר פחות, הקשב יותר                                                   │
│    ❌ להסביר את הרעיון 10 דקות                                            │
│    ✅ לשאול שאלה ולתת להם לדבר                                            │
│                                                                            │
│ ───────────────────────────────────────────────────────────────────────   │
│ 📊 המדד: אם אתה יוצא משיחה עם תחושה טובה על הרעיון -                     │
│         כנראה שאלת שאלות לא נכונות!                                       │
│         צריך לצאת עם עובדות, לא עם תחושות.                                │
└────────────────────────────────────────────────────────────────────────────┘
```

### שאלות טובות vs שאלות רעות

```
┌────────────────────────────────────────────────────────────────────────────┐
│ ✅ שאלות טובות (Mom Test Approved):                                       │
│                                                                            │
│ 🔍 על הבעיה:                                                              │
│ • "ספר לי על הפעם האחרונה ש-[הבעיה קרתה]"                                │
│ • "מה עשית כשזה קרה?"                                                     │
│ • "מה הכי מתסכל בזה?"                                                     │
│ • "כמה פעמים זה קורה לך?"                                                 │
│                                                                            │
│ 💰 על הערך:                                                                │
│ • "כמה זמן/כסף זה עולה לך היום?"                                         │
│ • "מה ניסית כבר כדי לפתור את זה?"                                        │
│ • "למה הפתרונות האלה לא עבדו?"                                           │
│                                                                            │
│ 🎯 על התנהגות:                                                             │
│ • "מתי פעם אחרונה חיפשת פתרון?"                                          │
│ • "מה גרם לך להפסיק לחפש?"                                               │
│ • "אם היית מוצא פתרון מושלם - איך היית יודע?"                            │
├────────────────────────────────────────────────────────────────────────────┤
│ ❌ שאלות רעות (Mom Test Fail):                                            │
│                                                                            │
│ • "מה אתה חושב על הרעיון שלי?" (דעה, לא עובדה)                           │
│ • "האם היית משתמש ב-X?" (עתיד, לא עבר)                                   │
│ • "האם תשלם 50₪ עבור זה?" (היפותטי)                                      │
│ • "נכון שזו בעיה גדולה?" (שאלה מנחה)                                     │
│ • "האם זה נשמע לך מעניין?" (מילוי חברתי)                                 │
└────────────────────────────────────────────────────────────────────────────┘
```

## 3.2 🧪 Pretotyping - בדיקה לפני בנייה

### הפילוסופיה

```
┌────────────────────────────────────────────────────────────────────────────┐
│ 🧪 Pretotyping (Alberto Savoia, Google):                                  │
│                                                                            │
│ 💡 Pretotype = Pretend + Prototype                                        │
│                                                                            │
│ 🎯 הרעיון:                                                                │
│ "Make sure you're building THE RIGHT IT                                   │
│  before you build IT RIGHT"                                               │
│                                                                            │
│ 📊 הסטטיסטיקה:                                                            │
│ • 90% מהסטארטאפים נכשלים                                                  │
│ • רוב הכישלונות = בנו משהו שאף אחד לא רצה                                │
│ • Pretotyping חוסך חודשים/שנים של בנייה מיותרת                           │
│                                                                            │
│ ⚡ ההבדל מ-Prototype:                                                      │
│ • Prototype = "האם אפשר לבנות את זה?"                                    │
│ • Pretotype = "האם כדאי לבנות את זה?"                                    │
│                                                                            │
│ ⏱️ זמן:                                                                    │
│ • Prototype = שבועות/חודשים                                               │
│ • Pretotype = שעות/ימים                                                   │
└────────────────────────────────────────────────────────────────────────────┘
```

### טכניקות Pretotyping

```
┌────────────────────────────────────────────────────────────────────────────┐
│ 🛠️ 6 טכניקות Pretotyping:                                                 │
│                                                                            │
│ 1️⃣ FAKE DOOR (דלת מזויפת)                                                 │
│    מה: כפתור/לינק/דף שמודיע "Coming Soon"                                │
│    מודד: כמה אנשים לחצו?                                                  │
│    דוגמה: כפתור "Buy Now" → "Thanks! We'll notify you"                   │
│    זמן: שעות                                                              │
│                                                                            │
│ 2️⃣ WIZARD OF OZ (קוסם מארץ עוץ)                                           │
│    מה: נראה אוטומטי, אבל אתה עושה ידנית מאחורה                           │
│    מודד: האם אנשים משתמשים? האם זה עובד?                                 │
│    דוגמה: "AI Recommendations" שאתה כותב ידנית                           │
│    זמן: ימים                                                              │
│                                                                            │
│ 3️⃣ CONCIERGE MVP                                                          │
│    מה: לתת את השירות ידנית ל-5-10 לקוחות                                 │
│    מודד: האם הם מוכנים לשלם? מה הם באמת צריכים?                         │
│    דוגמה: לתכנן אירועים ידנית לפני שבונים אפליקציה                       │
│    זמן: שבוע-שבועיים                                                      │
│                                                                            │
│ 4️⃣ SMOKE TEST (מבחן עשן)                                                  │
│    מה: Landing Page + פרסום → מודדים Sign-ups                            │
│    מודד: Conversion Rate, Cost per Lead                                  │
│    דוגמה: דף "הרשמה מוקדמת" עם תיאור המוצר                              │
│    זמן: ימים                                                              │
│                                                                            │
│ 5️⃣ ONE NIGHT STAND                                                        │
│    מה: לתת את המוצר לשימוש חד-פעמי                                       │
│    מודד: האם חוזרים? האם ממליצים?                                        │
│    דוגמה: גרסת נסיון ליום אחד                                            │
│    זמן: ימים                                                              │
│                                                                            │
│ 6️⃣ MECHANICAL TURK                                                        │
│    מה: אנשים עושים את מה שהמכונה "אמורה" לעשות                          │
│    מודד: האם התוצאה טובה? האם יש ביקוש?                                  │
│    דוגמה: "תרגום אוטומטי" שמתרגמים אנושיים עושים                        │
│    זמן: ימים-שבועות                                                       │
└────────────────────────────────────────────────────────────────────────────┘
```

### XYZ Hypothesis

```
┌────────────────────────────────────────────────────────────────────────────┐
│ 📊 XYZ Hypothesis - איך מנסחים השערה בדיקה:                               │
│                                                                            │
│ 📝 התבנית:                                                                 │
│ "לפחות X% מ-[קהל Y] יעשו [פעולה Z]"                                      │
│                                                                            │
│ 🎯 דוגמאות:                                                               │
│                                                                            │
│ EventApp:                                                                 │
│ "לפחות 5% מהמבקרים בדף הנחיתה ילחצו 'הרשמה מוקדמת'"                     │

│ "לפחות 50% מהזוגות שראיינתי יסכימו לשיחת המשך"                          │
│                                                                            │
│ ✅ XYZ טוב:                                                                │
│ • X = מספר ספציפי (לא "הרבה" או "מספיק")                                 │
│ • Y = קהל מוגדר וניתן למדידה                                             │
│ • Z = פעולה ניתנת למדידה (לא "יאהבו")                                    │
│                                                                            │
│ ❌ XYZ רע:                                                                 │
│ • "אנשים יאהבו את המוצר" (לא מדיד)                                       │
│ • "יהיה לנו הרבה משתמשים" (לא ספציפי)                                    │
│ • "אולי ישלמו" (לא מחייב)                                                │
│                                                                            │
│ 📈 מה עושים עם התוצאה:                                                    │
│ • עברנו את X → ממשיכים! ✅                                                │
│ • לא עברנו → בודקים למה, מחליטים Pivot או Stop                           │
└────────────────────────────────────────────────────────────────────────────┘
```

## 3.3 ✅ Testing - בדיקות משתמשים

### 5 Second Test

```
┌────────────────────────────────────────────────────────────────────────────┐
│ ⏱️ 5 Second Test:                                                          │
│                                                                            │
│ 🎯 מטרה: לבדוק אם המסר העיקרי ברור מייד                                  │
│                                                                            │
│ 📋 איך עושים:                                                              │
│ 1. הראה את המסך/דף למשתמש ל-5 שניות בדיוק                                │
│ 2. הסתר את המסך                                                           │
│ 3. שאל:                                                                    │
│    • "מה ראית?"                                                           │
│    • "מה החברה/המוצר עושה?"                                              │
│    • "למי זה מיועד?"                                                      │
│    • "מה הפעולה הבאה שהיית עושה?"                                        │
│                                                                            │
```
